SELinux en RHEL 9: Modos, Políticas y Troubleshooting Completo

Aprende a configurar SELinux en RHEL 9: enforcing mode, políticas, audit2allow y troubleshooting de errores de acceso denegado.

BY Andres Bernal (@abernal093) PUBLISHED 2026-03-30 STACKDEPLOY.DEV

Introducción

SELinux (Security-Enhanced Linux) es el sistema de control de acceso obligatorio (MAC) integrado en RHEL 9. A diferencia del modelo DAC tradicional de Linux, SELinux agrega una capa adicional de seguridad basada en políticas.

Modos de SELinux

SELinux opera en tres modos:

  • enforcing: Aplica la política y bloquea accesos no permitidos.
  • permissive: Solo registra violaciones sin bloquearlas. Útil para troubleshooting.
  • disabled: SELinux completamente deshabilitado (requiere reinicio).

Verificar el modo actual:

$ getenforce

Enforcing

$ sestatus

SELinux status: enabled

Current mode: enforcing

Loaded policy: targeted

Cambiar el Modo de SELinux

Cambio temporal:

setenforce 0 # permissive

setenforce 1 # enforcing

Cambio permanente en /etc/selinux/config:

SELINUX=enforcing

Contextos SELinux

Cada archivo y proceso tiene contexto: usuario:rol:tipo:nivel

$ ls -Z /var/www/html/

system_u:object_r:httpd_sys_content_t:s0 index.html

$ ps auxZ | grep httpd

system_u:system_r:httpd_t:s0 root 1234 httpd

Troubleshooting con audit2allow

$ ausearch -m avc -ts recent

$ ausearch -m avc -ts recent | audit2allow -M mipolitica

$ semodule -i mipolitica.pp

Gestión de Puertos con semanage

$ semanage port -l | grep http

http_port_t tcp 80, 443, 8008, 8009, 8443

$ semanage port -a -t http_port_t -p tcp 8080

Conclusión

SELinux es fundamental para la seguridad en RHEL 9. La clave es entender sus contextos y políticas para mantener el sistema seguro sin sacrificar funcionalidad.
